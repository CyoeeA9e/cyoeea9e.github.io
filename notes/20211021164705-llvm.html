1 LLVM IR

1.1 Identifiers

  • Global identifier（全局标识符）： @开头，函数、全局值(Global Value)
  • Local identifier（本地标识符）: %开头，寄存器名、类型

命名变量，使用字符串标识，未命名变量，使用无符号整数标识。 命名变量的运算过程中可能会产生未命名临时变量。 未命名变量的序号从0开始，基本块以及未命名函数的变量也会占用序号。

1.2 Module

LLVM程序由模块组成，每个模块都是输入程序的翻译单元。每个模块由函数、全局变量和符号表项组成。模块可以和LLVM链接器结合在一起，链接器会合并函数（和全局变量）的定义，解决前向声明，并合并符号表项。

Module由全局值，包括函数和全局变量(Global Variable)组成, 全局值通过指针来表示，并有一种Linkage Types

1.3 Linkage Types

  • private 全局值只能被同一个模块的对象访问。，并且不会在object文件的符号表中显示。
  • internal 和private近似，但值会在object文件的本地符号表中显示，类似C语言中的static关键字。
  • available_externally linkage类型为available_externally的全局值不会被释放到LLVM模块对应的对象文件中。从链接器的角度来看，available_externally全局相当于一个外部声明。它们的存在是为了让内联和其他优化在知道全局定义的情况下进行，而全局定义是在模块之外的某个地方。带有available_externally链接的全局允许被随意丢弃，并允许内联和其他优化。这种链接类型只允许在定义中使用，而不是在声明中。
  • linkonce
  • weak
  • common
  • appending
  • extern_weak
  • linkonce_odr, weak_odr
  • external

1.9 GetElementPtrInst

表示获取数据结构中的子元素操作

GetElementPtrInst的第一个索引i访问从该地址开始的第i个元素，因此如果需要访问地址处的元素，需要令i=0。

个人认为，这是为了避免产生混淆。如p[1]，访问的是p所在地址的第2个元素，还是p所在地址处的第2个成员，后者应该表示为p[0][1]

8 Metadata

8.1 Instruction

void Instruction::setMetadata(unsigned KindID, MDNode *Node) {
  if (!Node && !hasMetadata())
    return;

  // Handle 'dbg' as a special case since it is not stored in the hash table.
  if (KindID == LLVMContext::MD_dbg) {
    DbgLoc = DebugLoc(Node);
    return;
  }

  // Handle the case when we're adding/updating metadata on an instruction.
  if (Node) {
    auto &Info = getContext().pImpl->InstructionMetadata[this];
    assert(!Info.empty() == hasMetadataHashEntry() &&
           "HasMetadata bit is wonked");
    if (Info.empty())
      setHasMetadataHashEntry(true);
    Info.set(KindID, *Node);
    return;
  }

  // Otherwise, we're removing metadata from an instruction.
  assert((hasMetadataHashEntry() ==
          (getContext().pImpl->InstructionMetadata.count(this) > 0)) &&
         "HasMetadata bit out of date!");
  if (!hasMetadataHashEntry())
    return; // Nothing to remove!
  auto &Info = getContext().pImpl->InstructionMetadata[this];

  // Handle removal of an existing value.
  Info.erase(KindID);

  if (!Info.empty())
    return;

  getContext().pImpl->InstructionMetadata.erase(this);
  setHasMetadataHashEntry(false);
}
